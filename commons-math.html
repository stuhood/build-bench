Sample Results for Commons-Math

Compile and Tests, single-module project, 4 CPU laptop, as measured by GNU time. The value shown is the real (wall clock) time spend on the process, and the user-space CPU time (sum over all CPUs). The system-space CPU time was negligeable.

Clean build from scratch

Sbt is quickest, but cheating, it does not run all tests. Overall the other buildsystems doing parallelization show that on this hardware you cannot get below 140 seconds. Surprisingly pants took more time. This might be related to the great number of file operations.

Some buildsystems do not show any user CPU time, I assume this to be due to using separate daemon threads to do some work like nailgun.

Rebuild without changes

Several buildsystems recognize there is nothing to do, surprisingly pants is not among them.
